Build a new Flask app called "AI Governance Hub" with a very visual dashboard UI (Elastic/Kibana-like).

Hard requirements:
- Use Chart.js (CDN) + a modern CSS layout (cards, shadows, spacing, muted background, pill badges).
- The dashboard should feel like a real observability console: dense but readable, highly visual, minimal text.

Data source:
- DEFAULT_SOURCE_BASE_URL = "https://loan-decision-contract.replit.app"
- SOURCE_BASE_URL env var override
- /settings page to change base URL, stored in sqlite table settings(key,value)

Tabs:
1) Dashboard (primary)
2) Measure
3) Contract
4) Raw (Events/Decisions/Alerts)
5) Settings

Dashboard page must include (all visual):
A) KPI Card Row (large cards)
- Total (24h)
- Review Rate (24h)
- Human Overrides (7d)
- Slips (7d)
- Corrections (7d)
- Avg Risk (7d)
- Max Risk (7d)
- Pending Review Queue (current)

B) Charts (responsive grid, 2 columns):
1) Line chart: decisions per hour (last 24h) with series: approve/review/reject
2) Stacked bar: decisions per day (last 7d) approve/review/reject
3) Bar: top reason codes (last 7d)
4) Line: drift over time (last 7d) for avg credit score and avg DTI proxy
5) Gauge-like display (fake gauge is fine) for Review Rate vs baseline with thresholds

C) "Alerts & Drift" panel (right side)
- Show latest alerts from /api/alerts?limit=50 with severity pills
- Show drift deltas (credit/income/dti) as green/yellow/red depending on magnitude

D) "Recent Decisions" panel (table but polished)
- last 20 decisions with clickable decision link to source app /decision/<id>
- show decision pills, risk pill, engine version

Measure page (second tab):
- Same metrics but more detailed + definitions and calculation notes:
  slip, correction, override, review_rate, drift.
- Add a “Data quality” section:
  % decisions missing reasons_json
  % events missing decision_id
  count of DB errors (if tracked; else omit)

Contract page:
- Render /api/contract in a clean, readable layout with collapsible sections.

Raw page:
- Tables for /api/events, /api/decisions, /api/alerts with simple filters.

Implementation details:
- Add /api/proxy endpoints in Governance Hub that fetch from the source so the browser never hits CORS issues:
  /proxy/measure_summary?window=7d
  /proxy/alerts?limit=50
  /proxy/events?limit=200
  /proxy/decisions?limit=200
  /proxy/contract
- Use requests with timeout=5 and clear error messages.
- Use a single base template string with a nav bar and a dark-ish subtle background like observability tools.
- Use CSS variables for theme (bg, card, text, accent).
- Use Chart.js time-series with labels; if timestamps are ISO strings, parse them in JS.

Endpoints needed from source:
- /api/contract
- /api/measure/summary?window=24h|7d|all
- /api/events?limit=
- /api/decisions?limit=
- /api/alerts?limit=
If any are missing, show a friendly banner “Source missing endpoint: <url>”.

Deliverables:
- main.py
- requirements.txt
- governance.db (sqlite) created automatically
- No external build steps.
